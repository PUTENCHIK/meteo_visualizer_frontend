# Фронтенд проекта MeteoVisualizer

## О проекте

### Описание

Фронтенд фулстек-приложения визуализации измерений датчиков сети мачтовых комплексов МАМКА. Предоставляет доступ к функционалу приложения:

1. Система авторизации с хранением токенов в cookie.
2. Административные страницы для управление пользователями, их ролями и разрешениями ролей. Также создание в системе мачтовых комплексов МАМКА и связанных сущностей.
3. Просмотр визуализации измерений датчиков.

### Стек технологий

* **Среда выполнения**: Node.js v22.0.0+ и npm v10.0.0+
* **Фреймворк**: React
* **Язык**: TypeScript
* **Инструмент сборки**: Vite
* **Linting/Formatting**: ESLint и Prettier

## Развёртывание

### Общий старт

1. Клонирование репозитория:
    ```bash
    git clone https://github.com/PUTENCHIK/meteo_visualizer_frontend.git
    cd meteo_visualizer_frontend
    ```

2. Настройка окружения. Скопировать `.env.example` или `.env.docker.example`, в зависисмости от способа поднятия, и переименовать в `.env` и `.env.docker`. В качестве окружения будет использоваться соответствующий файл. Поменять переменные окружения:

    * `VITE_API_URL`. URL бэкенда приложения. Поменять порт, если он менялся при развёртывании бэкенда.
    * `VITE_APP_PORT`. Порт самого приложения. При смене, также поменять порт в `ALLOW_ORIGINS` бэкенда.

### Локально

1. При необходимости, установить Node.js и npm:
```bash
# Используя nvm
nvm install 22
nvm use 22

# Или напрямую
sudo apt update
sudo apt install nodejs npm
```

2. Установить зависимости:
    ```bash
    npm install
    ```

3. Запуск в режиме разработки:
    ```bash
    npm run dev
    ```

### Docker

1. После настройки `.env.docker`, поднять контейнер:

    ```bash
    docker compose --env-file .env.docker up -d --build
    ```

## Структура проекта

```
.
├── src/
│   ├── assets/                 # SVG-иконки, используемые в коде
│   ├── components/
│   │   ├── dialogs/            # Диалоговые окна
│   │   ├── entity-items/       # Компоненты-карточки, отображающие разные сущности приложения
│   │   ├── forms/              # Формы (компоненты и zod-схемы)
│   │   ├── helpers/            # Вспомогательные компоненты без контента
│   │   ├── panels/             # Передвигаемые окна на странице комплекса
│   │   ├── three-models/       # Модели R3F
│   │   ├── ui-components/      # Общие UI компоненты форм и вёрстки (в основном)
│   ├── context/                # Контексты и провайдеры приложения
│   ├── hooks/                  # Пользовательские хуки
│   ├── managers/               # Менеджеры-синглтоны
│   ├── pages/                  # Страницы с связанные с ними компоненты вёрстки
│   ├── stores/                 # Сторы (zustand и другие)
│   ├── styles/                 # статичные файлы стилей
│   ├── utils/                  # Вспомогательные общие функции, интерфейсы и типы
│   ├── axios.d.ts              # Дополнительные инструменты
│   ├── vite-env.d.ts           # Дополнительные инструменты
│   └── main.tsx                # Класс для загрузки .env
├── public/                     # Публичные статичные файлы
├── .env                        # Файл локального окружения (игнорируется Git)
├── .env.docker                 # Файл окружения Docker (игнорируется Git)
├── .env.example                # Пример файла локального окружения
├── .env.docker.example         # Пример файла окружения Docker
├── .editorconfig               # Конфигурация редакторов кода
├── .prettierrc.js              # Настройки форматтера Prettier
├── eslint.config.js            # Описание правил линтера ESLint
├── package-lock.json           # Фиксация точных версий зависимостей (генерируется автоматически)
├── package.json                # Манифест проекта: список зависимостей, скрипты запуска и метаданные
├── tsconfig.json               # Главный конфиг TypeScript (подключает 2 других)
├── tsconfig.app.json           # Специфичные настройки TS для кода самого приложения
├── tsconfig.node.json          # Настройки TS для файлов окружения 
├── vite.config.ts              # Конфигурация сборщика Vite
├── docker-compose.yml          # Описание инфраструктуры Docker
└── Dockerfile                  # Инструкция для сборки образа приложения
```

## Разработка

Запуск форматирования и линтера:
```bash
npm run lint
```

Запуск с фиксом известных проблем:
```bash
npm run lint:fix
```
